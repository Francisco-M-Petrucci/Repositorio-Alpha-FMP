<summary>## CONCEITOS BÁSICOS</summary>
* **Redundância:** Duplicação de componentes ou funções críticas para aumentar a confiabilidade.
* **Confidencialidade:** Garantia de que a informação é acessível apenas a pessoas autorizadas.
* **Integridade:** Assegura que os dados não sofreram alterações não autorizadas.
* **Autenticidade:** Confirmação da identidade de quem envia a informação ou da fonte.
* **Resiliência:** Capacidade de um sistema se recuperar rapidamente de falhas.
* **Escalabilidade:** Capacidade de um sistema crescer e gerenciar aumento de demanda.
* **Machine Learning:** Subcampo da IA onde sistemas aprendem com base em dados.
* **Bits:** A menor unidade de informação em computação (0 ou 1).
* **Pacotes de Dados:** Unidades formatadas de dados enviadas através de uma rede.
* **Experiência do Usuário (UX):** Como uma pessoa se sente ao interagir com um sistema.
* **Frontend / Backend:** Parte visual do usuário e a lógica do servidor, respectivamente.
* **Banco de dados:** Coleção organizada de informações estruturadas.
* **Tráfego:** Fluxo de dados movendo-se através de uma rede.
* **Monitoramento:** Acompanhamento contínuo da saúde e performance do sistema.
* **Logs:** Registros de eventos que ocorrem dentro de um sistema ou software.
</details>

<details>
<summary>## SEGURANÇA</summary>
* **Autenticação:** Processo de verificar a identidade de um usuário.
* **Ataques:** Tentativas maliciosas de comprometer sistemas.
* **Malware:** Software projetado para causar danos a computadores.
* **LGPD:** Lei Geral de Proteção de Dados Pessoais brasileira.
* **Criptografia:** Prática de codificar informações para impedir acesso não autorizado.
    * **Simétrica:** Usa a mesma chave para criptografar e descriptografar.
    * **Assimétrica:** Usa um par de chaves (pública e privada).
* **VPN:** Rede Privada Virtual para conexão segura.
    * **Comercial:** Serviços para privacidade do usuário final.
    * **Acesso Remoto:** Conexão de usuários externos à rede corporativa.
    * **Site-to-site:** Conexão entre redes inteiras (ex: matriz e filial).
* **Túnel de comunicação:** Encapsulamento de dados para transmissão segura.
* **Dados Sensíveis:** Informações que exigem proteção rigorosa (ex: financeiros).
* **Ataque DoS / DDoS:** Negação de serviço para tornar um recurso indisponível.
* **Botnets:** Redes de dispositivos infectados controlados remotamente.
* **Limite de Requisições:** Restrição do número de solicitações de um usuário.
* **Antivírus:** Software para prevenir e remover malware.
* **Firewalls:** Dispositivos que filtram o tráfego de rede.
    * **De Pacotes:** Filtra baseado em cabeçalhos (IP, porta).
    * **Circuit Level Gateways:** Verifica o handshake TCP.
    * **Stateful Inspection Firewalls:** Monitora o estado das conexões.
    * **Firewalls de aplicação:** Filtra na camada de aplicação (WAF).
    * **Firewalls de próxima geração:** Combina firewall tradicional com inspeção profunda.
* **DNS Cache Poisoning:** Inserção de dados falsos no cache DNS.
* **DNSSEC:** Extensão de segurança para o DNS.
* **Randomização da porta de origem:** Dificulta ataques de falsificação de DNS.
* **SOAR:** Orquestração de segurança e resposta a incidentes.
* **E2EE:** Criptografia de ponta a ponta.
* **Insider Threat:** Ameaça vinda de dentro da organização.
* **Princípio de Menor Privilégio:** Acesso mínimo necessário para a função.
* **WPA3:** Padrão de segurança Wi-Fi atual.
* **Sniffing:** Monitoramento de dados passando pela rede.
* **Man-in-the-middle:** Interceptação da comunicação entre duas partes.
* **Spoofing:** Falsificação de identidade para ganhar confiança.
* **Ataques por Engenharia Social:** Manipulação psicológica de pessoas.
* **Vulnerabilidades:** Falhas de segurança (Backdoors, Exploits).
* **Movimentação Lateral:** Movimento do atacante dentro da rede após invasão.
* **Gestão de Patches:** Aplicação de atualizações de correção.
* **Autoridades Certificadoras:** Emissores de certificados digitais confiáveis.
* **Time To Live (TTL):** Tempo de vida de um pacote na rede.
* **Hashes:** Impressão digital de dados de tamanho fixo.
</details>

<summary>## ESTRUTURA DA INTERNET</summary>
* **HTML:** Linguagem de marcação para páginas web.
* **URL:** Endereço que localiza recursos na internet.
* **Topologias de Rede:** Layout das conexões (Malha, Estrela, Anel, Barramento).
* **Backbones:** As rotas principais da internet.
* **NAT:** Traduz IPs privados em públicos.
* **ISPs:** Provedores de acesso à internet.
* **Blockchain:** Livro-razão digital descentralizado.
    * **Princípio de Consenso:** Validação de transações na rede.
* **Web 2.0 / 3.0:** Web social vs. Web semântica/descentralizada.
* **Single Page Applications (SPA):** Aplicação web de página única dinâmica.
* **Internet, Intranet, Extranet:** Rede global, interna, e estendida a parceiros.
* **IPv4, IPv6:** Versões de endereçamento IP (32 vs 128 bits).
* **Conexão peer-to-peer:** Rede descentralizada ponto a ponto.
* **IoT:** Internet das Coisas.
* **Portas (Gateways):** Pontos de conexão entre redes.
* **Máscara de Rede (/24):** Define a parte da rede e do host no IP.
* **Endereço Público/Privado:** IP global vs. IP local.
* **Ping:** Teste de alcançabilidade de host.
* **Traceroute:** Rastreio da rota do pacote.
* **Webservices:** Serviços de comunicação entre aplicações.
* **AJAX:** Atualização assíncrona de páginas web.
* **Microsserviços:** Arquitetura de serviços pequenos e independentes.
* **DNS:** Tradução de nomes em IPs.
* **DNS Dinâmico:** Atualiza o IP do host automaticamente.
* **DHCP:** Atribuição dinâmica de IPs.
* **Servidores Autoritativos:** Detentores da resposta final do domínio.
* **Query ID:** Identificador da consulta DNS.
* **CDN:** Rede de distribuição de conteúdo.
* **PoP:** Ponto de presença física da rede.
* **PTR:** Registro DNS reverso (IP para nome).
* **ARPA:** Domínio de infraestrutura técnica.
* **Regras para Registro de Domínio:** Diretrizes de nomes de sites.
* **CGI.br:** Comitê Gestor da Internet no Brasil.
* **Registrars:** Vendedores de domínios.
* **Ethernet:** Padrão para redes cabeadas.
* **Token Ring:** Tecnologia de rede baseada em token (antiga).
* **OSI:** Modelo de 7 camadas de rede.
    * **Application:** Interface com o usuário.
    * **Presentation:** Formatação de dados.
    * **Session:** Gerenciamento de sessões.
    * **Transport:** Entrega confiável (TCP/UDP).
    * **Network:** Roteamento (IP).
    * **Data Link:** Transferência física (MAC).
    * **Physical:** Hardware e cabos.
* **TCP/IP:** Modelo de protocolos da internet.
* **3G, 4G, 5G:** Gerações de rede móvel.
* **Bluetooth:** Comunicação sem fio de curto alcance.
* **Masking:** Ocultação de dados.
* **Cloudflare:** Serviços de CDN e segurança.
* **Amazon AWS:** Computação em nuvem da Amazon.
* **Amazon S3:** Armazenamento de objetos na nuvem.
</details>

<summary>## SERVIDORES E REDES</summary>
* **Proxies:** Intermediários entre cliente e servidor.
* **Roteadores:** Encaminham pacotes entre redes.
* **Switches:** Conectam dispositivos na mesma rede.
* **Balanço de Carga:** Distribuição de tráfego.
    * **Round Robin, Weighted, GSLB:** Métodos de distribuição.
* **Quality of Service (QoS):** Priorização de tráfego crítico.
* **Bandwidth:** Largura de banda da conexão.
* **Latência:** Tempo de atraso na rede.
* **Jitter:** Variação da latência.
* **API:** Interface de comunicação entre softwares.
* **JSON / XML / YAML:** Formatos de dados.
* **Broadcast, Multicast, Anycast:** Modos de transmissão de dados.
* **CNAME:** Apelido para nome de domínio.
* **Alta Disponibilidade:** Sistemas operando sem falhas.
* **Gateway Failover:** Link de backup automático.
* **LAG / LACP:** Agregação de links.
* **ECMP:** Roteamento de múltiplos caminhos iguais.
* **MPLS:** Encaminhamento baseado em labels.
* **PBR:** Roteamento baseado em políticas.
* **LAN, WAN, MAN, PAN:** Tipos de rede por abrangência.
* **Segmentação de Redes:** Divisão para segurança.
* **Sub-redes:** Divisão lógica de IPs.
* **Arquitetura SDN:** Rede definida por software.
* **Infraestrutura On Premises:** Servidores locais.
* **Infraestrutura Cloud:** Servidores na nuvem.
</details>

<summary>## PROTOCOLOS</summary>
* **Cabeçalhos:** Metadados do pacote.
* **UDP / TCP:** Protocolos de transporte (rápido vs. confiável).
* **RFC:** Documentos de padrões da internet.
* **SSL/TLS:** Criptografia de transporte.
    * **TCP Handshake:** Início da conexão.
    * **TCP SYNC/ACK:** Pacotes de sincronização.
    * **Client/Server Hello:** Negociação TLS.
    * **Certificado SSL:** Identidade do site.
    * **IKE:** Troca de chaves.
    * **Session Key:** Chave temporária da sessão.
* **WPA / WEP:** Segurança Wi-Fi.
* **IPsec:** Segurança para IP (VPNs).
* **REST:** Arquitetura para web services.
* **SOAP:** Protocolo baseado em XML.
* **OAuth:** Autorização de acesso.
* **VOIP:** Voz sobre IP.
* **BGP:** Roteamento da internet global.
* **OSPF:** Roteamento interno (estado do link).
* **RIP:** Roteamento por saltos (antigo).
* **HTTP/HTTPS:** Protocolo web (seguro).
* **LoRaWAN:** Rede IoT de longa distância.
* **Zigbee:** Rede IoT em malha.
</details>
